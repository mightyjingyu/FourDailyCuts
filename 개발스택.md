# 싶개(SIPGAE) 서비스 통합 개발 문서

본 문서는 현재 코드베이스를 기준으로, 서비스 구현에 사용된 기술, 핵심 로직, 코드 자체 평가, UI/UX 설계 의도, 운영 관점의 서비스 기획 요소를 통합 정리한 문서입니다.

---

## 1) 서비스 개요

### 1-1. 서비스 한 줄 정의
- "강아지 감성 테마 프레임으로 4컷 사진을 자동 촬영하고 바로 저장하는 웹 포토부스"

### 1-2. 핵심 가치
- 설치 없이 웹에서 즉시 촬영
- 촬영-보정-프레임 합성-저장까지 한 번에 제공
- 감성형 테마(싶개 프레임) + 미니멀 테마(기본 프레임) 동시 제공

### 1-3. 주요 기능 요약
- 홈/선택/촬영/완료 단일 플로우
- 6개 테마 프레임 제공 (`green`, `yellow`, `purple`, `red`, `basicBlack`, `basicWhite`)
- 4장 자동 타이머 촬영
- 얼굴 랜드마크 기반 뷰티 워프(얼굴 슬림 + 눈 확대)
- 최종 프레임 PNG 저장

## 2) 개발 스택 (Tech Stack)

### 2-1. 프레임워크/언어
- `Next.js 15.3.5` (App Router 구조)
- `React 19.2.0`
- `TypeScript 5.x` (`strict: true`)

### 2-2. 주요 라이브러리
- `@mediapipe/face_mesh`
  - 얼굴 랜드마크 검출
  - 실시간 프레임 보정 로직의 입력 데이터 제공
- `html2canvas`
  - 프레임 DOM을 고해상도 이미지로 캡처하여 저장

### 2-3. 스타일/리소스
- 전역 CSS 기반 스타일링: `src/app/globals.css`
- 테마별 프레임 색상, 장식, 애니메이션, 타이포 제어
- 퍼블릭 이미지 리소스(`public/`) 활용

### 2-4. 개발 도구/품질
- ESLint (`eslint-config-next`)
- TypeScript strict 모드
- npm 스크립트
  - `npm run dev`
  - `npm run build`
  - `npm run start`
  - `npm run lint`

### 2-5. 브라우저 API
- `navigator.mediaDevices.getUserMedia` (카메라 스트림)
- `HTMLCanvasElement` 2D context (프리뷰 렌더, 캡처, 필터, 워프)
- `requestAnimationFrame` (실시간 프리뷰 루프)
- `setInterval`, `setTimeout` (촬영 타이머/화면 전환)

## 4) 핵심 로직 설계

### 4-1. 단계형 상태 머신
`SipgaeApp`의 `step` 상태로 흐름 제어:
- `home`: 랜딩 및 시작
- `select`: 프레임 선택
- `loading`: 전환 대기
- `shoot`: 카메라 촬영
- `done`: 결과 확인 및 저장

장점:
- 화면 전환이 단순하고 예측 가능
- 단일 컴포넌트에서도 전체 흐름 파악이 쉬움

### 4-2. 프레임 선택 로직
- 테마 옵션을 "싶개 프레임"과 "기본 프레임"으로 분리 표시
- 좌우 버튼으로 캐러셀 인덱스를 이동
- 선택 시 하이라이트 처리

### 4-3. 결제(데모) 처리
- 양심 결제 모달은 현재 데모 구조
- `입금 완료` 클릭 시 바로 촬영 플로우 진입
- 실제 PG/검증 로직은 아직 미적용

### 4-4. 카메라 프리뷰/보정/캡처
- `step === "shoot"`일 때만 카메라 스트림 시작
- 종료/이동 시 스트림 및 FaceMesh 인스턴스 정리
- 실시간 프리뷰는 canvas 기반으로 렌더
  - 미러링
  - 필터 적용
  - 랜드마크 기반 부분 워프

### 4-5. 얼굴 보정 알고리즘 요약
- FaceMesh 결과를 주기적으로 받아 `landmarksRef`에 저장
- 얼굴 중심/폭/높이를 계산해 영역 워프
  - V-line: 얼굴 폭을 소폭 축소
  - Eye enlargement: 양쪽 눈 주변 확대
- 과도한 왜곡을 막기 위해 clip/alpha 기반의 완만한 합성 사용

### 4-6. 자동 촬영 로직
- 일정 주기(7초)로 자동 촬영
- `photos` 배열의 첫 빈 슬롯부터 순차 삽입
- 4장 채워지면 자동으로 `done` 이동

### 4-7. 결과 저장 로직
- `html2canvas(frameRef.current, { scale: 2, useCORS: true })`
- 애니메이션이 결과물에 흔들림을 주지 않도록 저장 직전 정적 모드 적용
- `canvas.toBlob` 후 다운로드 링크 생성

## 5) UI 설계 정리

### 5-1. 시각 콘셉트
- "강아지 + 스티커 + 포토부스" 감성 기반
- 테마별 톤앤매너를 CSS 클래스 단위로 분리
- 기본 프레임은 실사용 포토부스 느낌으로 단정하게 구성

### 5-2. 프레임 디자인 시스템
- 프레임 크기 고정형 카드 + 4개 사진 슬롯
- 슬롯 비율은 `4:3` 유지
- 최근 수정 반영:
  - 슬롯 모서리 직사각형화
  - 슬롯 간 간격 축소
  - 선택 미리보기 슬롯 배경 흰색 고정
  - 기본 흰/검정 프레임 외곽선 명확화

### 5-3. 타이포그래피
- 하단 타이틀 가독성 강화를 위해:
  - 글자 크기 확대
  - stroke/shadow 강화
  - 애니메이션 완화 또는 제거
- 테마별 컬러 가독성 보정 추가

## 6) UX 설계 정리

### 6-1. 사용자 여정
1. 홈에서 시작
2. 프레임 선택
3. 결제 확인(데모)
4. 자동 촬영 4회
5. 결과 확인 후 저장

### 6-2. UX 의도
- "클릭 최소화": 촬영은 자동 진행
- "실패 최소화": 카메라 오류 메시지 즉시 안내
- "완료 중심": 촬영 완료 후 바로 저장 CTA 노출

### 6-3. 피드백 디자인
- 촬영 카운트다운
- 찰칵 피드백(플래시/문구)
- 로딩 상태 표시

### 6-4. 접근성/호환성 관점
- 버튼에 `aria-label` 일부 적용
- 카메라 권한 정책은 브라우저 보안 정책 영향
- iOS Safari 등 모바일 브라우저 테스트 확대 필요

## 7) 코드 자체 평가 (Self Review)

### 7-1. 강점
- 컴포넌트 분리와 역할 구분이 명확함 (`SipgaeApp` vs `PhotoFrame`)
- 상태 전이가 단순해 디버깅이 쉬움
- 실시간 보정 + 저장까지 완결된 사용자 경험 제공
- CSS 테마 확장성이 높아 프레임 추가가 쉬움

### 7-2. 개선이 필요한 지점
- `SipgaeApp.tsx` 파일이 커져 단일 파일 복잡도가 높음
- 스타일이 전역 CSS에 집중되어 변경 영향 범위가 큼
- 결제 로직이 데모 상태라 서비스 완성도 한계 존재
- 자동 촬영/보정 파라미터가 하드코딩되어 운영 조정이 어려움

### 7-3. 리팩토링 제안
- 촬영 엔진 훅 분리 (`useCameraCapture`, `useFaceBeauty`)
- 단계별 뷰 컴포넌트 분리 (`HomeView`, `SelectView`, `ShootView`, `DoneView`)
- 테마 메타데이터 JSON화 및 타입 분리
- 보정 강도/타이머를 설정값으로 분리

## 11) 배포 전 꼭 체크해야 할 3가지 (중요)

### ① HTTPS 환경 확인 (카메라 권한)
- 웹캠(`getUserMedia`) API는 보안상 HTTPS 환경에서만 정상 작동합니다.
- Vercel은 기본 HTTPS를 제공하므로 배포 자체는 문제없습니다.
- 다만 로컬 테스트와 달리 실제 배포 후 모바일 기기에서 카메라 권한 팝업이 정상 노출되는지 꼭 확인합니다.

### ② 환경 변수 (Environment Variables)
- 외부 API 키, 비공개 설정값은 `.env` 파일을 저장소에 올리지 않습니다.
- Vercel `Project Settings > Environment Variables`에 직접 등록해야 합니다.
- 현재 프로젝트는 데모 결제 기준이므로 즉시 필수값은 많지 않지만, 실결제/외부 API 연동 시 필수 체크 항목입니다.

### ③ html2canvas와 외부 이미지(CORS) 이슈
- 현재 저장 로직은 `useCORS: true`를 사용합니다.
- `public/` 폴더 외부 URL 이미지를 쓰면 배포 환경에서 캔버스 오염(Tainted Canvas)으로 저장 이미지에 누락이 생길 수 있습니다.
- 프레임 에셋은 가능한 `public/` 내부 정적 자산으로 통일하고, 외부 이미지 사용 시 CORS 헤더를 반드시 확인합니다.
